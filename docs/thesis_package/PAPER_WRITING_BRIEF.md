# 论文初稿写作资料包说明

## 论文建议题目

基于多智能体协作的自动化代码生成与审查系统的设计与实现

## 项目定位

本项目不是普通聊天网页，而是一个可进化的多智能体软件开发与个人助理系统。系统围绕自然语言软件开发任务，构建了 Agent Run、Agent Registry、Capability Registry、长期记忆、Transcript 可观测事件流、动态 Tool 配置和前端可视化演示台。

## 写作主线

建议论文按以下逻辑展开：

1. 绪论
   - 大模型辅助开发的背景
   - 单一聊天式代码生成的不足：缺少任务拆解、工具调用、长期上下文、过程可观测和结果审查
   - 本系统目标：构建可扩展、可观测、可演示的多智能体代码生成与审查平台

2. 相关技术
   - 大语言模型与 Agent
   - 多智能体协作
   - FastAPI / React 前后端分离
   - 长期记忆与检索增强
   - 事件驱动与工作流编排

3. 需求分析
   - 自然语言任务提交
   - 多 Agent 协作执行
   - 工具能力扩展
   - 记忆检索与上下文注入
   - Run/Transcript 过程追踪
   - 前端演示和可视化

4. 系统总体设计
   - React 前端
   - FastAPI 后端
   - Agent Registry
   - Capability / Tool Registry
   - Memory 模块
   - Workflow / Agent Run
   - Transcript JSONL 事件流
   - YAML 配置和热重载

5. 核心模块设计与实现
   - Agent 管理模块
   - 多智能体运行模块
   - 能力插件与动态 Tool 模块
   - 长期记忆模块
   - Transcript 可观测模块
   - 前端运行演示台

6. 系统测试与演示
   - 健康检查
   - Agent 注册状态
   - 一键 Demo 任务
   - Run 创建、执行、完成
   - Transcript 时间线
   - 最终输出展示

7. 总结与展望
   - 已完成能力
   - 不足：认证、持久化、多 worker、真实 MCP 深度集成、前端自动化测试等
   - 展望：更完整的工具生态、离线回放、多用户权限、性能优化

## 不要乱写的点

- 不要说系统已经实现了完整生产级权限认证；当前没有用户鉴权。
- 不要说支持多 worker 分布式部署；当前主要是单进程运行。
- 不要把底层 `thinking` 事件写成“模型隐藏思维”；论文里统一写“流式生成片段”或“事件流片段”。
- 不要夸张说完全自动替代程序员；应写成辅助软件开发、代码生成、审查和演示验证。
- Mermaid 图可以作为论文图的草稿，正式论文可重新绘制为 Visio/draw.io 风格。

## 可直接引用的创新点

1. 多智能体编排：Assistant 作为主控，Planner/Coder/Reviewer 可被委派。
2. Agent-as-Tool：将 Agent 封装为能力，统一进入 Capability Registry。
3. 工具能力扩展：通过配置和动态 Tool 扩展系统能力。
4. 长期记忆：支持记忆形成、存储、检索和上下文注入。
5. 运行轨迹可观测：Run 的生命周期事件写入 transcript，并在前端以时间线展示。
6. 配置驱动与热重载：Agent、Tool、Workflow 等通过 YAML 配置组织，降低扩展成本。
